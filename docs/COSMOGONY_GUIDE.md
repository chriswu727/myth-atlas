# 创世时间线 撰写规范

每个有创世叙事的体系一个文件：`data/cosmogony/<traditionId>.json`。
写完必须跑 `node scripts/validate.mjs` 到零 error。

## 它是什么

**神话内部的叙事时间线**，不是历史年表。讲的是「这个文明认为世界是怎么开始的」，按神话自身的先后顺序排：太初有什么 → 天地怎么分开 → 世界怎么成形 → 人怎么来的 → 后来出了什么事。

每一步要挂一个 `motif`（母题），这是跨文明对照页的对齐轴——正因为有它，读者才能横向看到：华夏的「盘古垂死化身」和北欧的「尤弥尔尸体造世界」是同一个母题的两种长法。

## Schema

```json
{
  "tradition": "chinese",
  "source": { "zh": "《三五历纪》《淮南子》《风俗通义》", "en": "Sanwu Liji, Huainanzi, Fengsu Tongyi" },
  "note": {
    "zh": "华夏没有单一定本的创世叙事，盘古故事晚出（三国），女娲造人见于汉代文献，此处按后世通行的整合顺序排列。",
    "en": "..."
  },
  "stages": [
    {
      "motif": "chaos",
      "phase": { "zh": "太初", "en": "In the beginning" },
      "title": { "zh": "混沌如鸡子", "en": "Chaos like an egg" },
      "text": { "zh": "两到四句话，讲清这一步发生了什么。要有具体细节。", "en": "..." },
      "entries": ["pangu"]
    }
  ]
}
```

### 字段

| 字段 | 必填 | 说明 |
|---|---|---|
| `tradition` | ✅ | 必须与文件名一致，且是 traditions.json 里已注册的 id |
| `source` | ✅ | 这条时间线依据的文献，双语。**严禁编造** |
| `note` | 可选 | 需要说明的地方：有多个互相冲突的版本、叙事晚出、是学者整合的、原文残缺等。**有争议就一定要写**，这是诚实度所在 |
| `stages` | ✅ | 3–8 步。少于 3 步说明没讲清楚，多于 8 步就太碎 |
| `stages[].motif` | ✅ | 见下方母题表，用于跨文明对齐 |
| `stages[].phase` | ✅ | 该步在本体系里的阶段名，短，双语（如「太初」「开天辟地」） |
| `stages[].title` | ✅ | 这一步的标题，一句话概括，双语（如「混沌如鸡子」） |
| `stages[].text` | ✅ | zh ≥ 60 字，en ≥ 30 词。讲清事件，用具体细节（谁、用什么、变成了什么） |
| `stages[].entries` | 可选 | 相关条目 id，软引用（指向不存在的条目只 warn） |

### 母题表（motif 枚举）

| motif | 中文 | 含义 |
|---|---|---|
| `chaos` | 太初 | 世界开始之前的状态：混沌、深渊、原初之水、虚无 |
| `first-beings` | 最初的存在 | 第一批神或原初存在的出现 |
| `separation` | 天地分离 | 天与地（或水与陆）被分开、撑开、推离 |
| `world-form` | 世界成形 | 世界的实体如何造出：巨人尸体、宇宙卵、潜水取土、言语创造 |
| `humans` | 人类诞生 | 人从什么造出来、由谁造 |
| `ordeal` | 洪水与劫 | 大洪水、诸神之战、纪元更替、人类被毁灭重造 |
| `now` | 当世 | 现在这个世界/纪元如何确立，以及（若有）它将如何终结 |

一个体系不必集齐所有母题——**缺哪个母题本身就是信息**（对照页会显示为空，读者一眼看出「凯尔特没有创世叙事」「希腊没有造人的正典版本」）。同一个 motif 也可以在一个体系里出现两次（如玛雅三次造人失败）。

## 写作要求

- 双语各自地道，不是互译。
- 用具体细节说话：盘古身高日长一丈，不是「盘古很高大」。
- 禁 AI 腔：不写「不是X而是Y」论证句、不写空洞拔高、不写总结性抒情。
- 禁 emoji 和装饰性符号。
- 有争议、有多个版本、叙事晚出的，写进 `note`，不要在正文里假装只有一个定本。
- 活态信仰（圣经传统、原住民传统）按「传统叙事」客观陈述，不加评判。
- **严禁编造**文献、细节、年代。不确定就不写。

## 没有创世叙事的体系怎么办

不是每个体系都有。山海经是博物志不是创世书，日本怪谈和美洲怪谈是都市传说，失落之地是地理传说——这些**不建文件**，页面上不显示时间线。

但如果一个体系确实缺乏完整创世叙事、而这件事本身值得说（如凯尔特：爱尔兰神话没有创世，只有《夺取爱尔兰记》的入侵序列），可以建文件，用 `note` 把这个情况讲清楚，`stages` 写它实际有的那套开端叙事。诚实地说「这里没有创世神话」比硬编一个出来有价值得多。
